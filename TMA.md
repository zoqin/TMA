# matrice eisenhower
## urgent et important
- création entité tâche
- création de test unitaire tâche

## urgent mais pas important
- Création d'un trello
- Mise en place d'un slack pour communiquer en temps réel

## Important mais pas urgent
- création de teste d'intégration
- création de teste fonctionnel

## Pas important pas urgent
- Développer des feature de gestion de tâche
- duplication de tâche
- organisation des tache par importance

# Maintenance applicative
## Test unitaire
Veuillez créer des test unitaire pour tout le code que vous produisez c'est à dire un teste par Entité, par Service
Tester toute vos méthode et vos fonctions

## Test d'intégration
Compléter les test d'intégration si possible avec les traitement pertinent qui consernes votre travail

## Test fonctionnel
Nous conceverons plus tard dans le développement des test fonctionnelle pour vérifié le parcours client

# Processus de traitement d'un incident
Veuillez ouvrir un ticket et une tâche trello aussitôt, prevenez les développeur concerné par la feature impacté
Documenté si l'incident était couvert par des test 

# gestion des demande d'évolution
Tout les erreurs impactant le fonctionnement au point de vue client sont à classé dans urgent et important
Les autres sont à débattre dans le slack
Pour les features complémentaire ce n'est pas une priorité dans l'état actuelle de l'application mais seront tracké pour de future implémentation
